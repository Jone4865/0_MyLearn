1) 서버 사이드 렌더링은 클라이언트(브라우저)가 서버에 매번 데이터를 요청하여 서버에서 처리하는 방식입니다. 
클라이언트에서 요청이 들어올 때마다 매번 서버에서 새로운 화면을 만들어 제공하는 것입니다. 
서버로부터 완전하게 만들어진 html 파일을 받아와 페이지 전체를 렌더링 하는 방식이다

2) 서버 사이드 렌더링은 서버에서 페이지를 그려 클라이언트로 보낸 후 화면에 표시하는 기법이며, 
첫 렌더링된 html을 클라이언트에 전달해주기 때문에 초기 로딩 속도를 많이 줄여줄 수 있습니다. 
단점은 프로젝트의 복잡도가 증가하고 페이지 이동시 화면이 깜빡거리는 형상이 발생할 수 있습니다.(페이지 요청마다 페이지 새로고침의 발생)

SSR과 CSR의 차이
표시해줄 화면을 어디서, 어떻게 그리냐의 차이이다.
CSR은 페이지의 내용을 브라우저에서, 그리고 SSR은 서버에서 페이지의 내용을 다 그려서 브라우저로 던져준다
서버 사이드 렌더링(SSR) : 클라이언트(브라우저)가 서버에 매번 데이터를 요청하여 서버에서 처리하는 방식입니다. 
클라이언트에서 요청이 들어올 때마다 매번 서버에서 새로운 화면을 만들어 제공하는 것입니다. 
단점 :  
1) Blinking issue 사용자가 새로고침을 하게 되면 전체 웹사이트를 다시 서버에서 받아와야 하기 때문에 화면이 없어졌다가 나타난다 (깜빡임 현상)
2) 서버 과부하가 걸리기 쉽다는 점이다. 사용자가 많은 제품일수록 서버에 데이터를 요청하는 횟수가 많아지기 때문이다
